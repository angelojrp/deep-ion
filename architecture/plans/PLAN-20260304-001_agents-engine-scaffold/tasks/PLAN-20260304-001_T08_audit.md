### Especificação Técnica

#### `audit/decision_record.py`

```python
from datetime import datetime, timezone
from typing import Literal
from pydantic import BaseModel, ConfigDict, Field

DecisionType = Literal["approve", "block", "escalar", "alert"]

class DecisionRecord(BaseModel):
    model_config = ConfigDict(frozen=True)

    skill_id: str                              # ex: "SKILL-DISC-00"
    timestamp_utc: datetime = Field(
        default_factory=lambda: datetime.now(tz=timezone.utc)
    )
    decision: DecisionType
    confidence_score: float                    # 0.0 – 1.0
    justification: str
    rn_triggered: tuple[str, ...] = ()         # ex: ("RN-03", "RN-07")
    model_used: str                            # model id ou "deterministic"
    issue_number: int | None = None            # issue GitHub vinculada
```

**Regras imutáveis do schema:**
- `frozen=True` — instâncias não mutáveis após criação
- `rn_triggered` como `tuple` (não `list`) para garantir hashabilidade e imutabilidade
- `confidence_score` deve ser validado no intervalo `[0.0, 1.0]` via `Field(ge=0.0, le=1.0)`
- Timestamps sempre em UTC (`timezone.utc`)

#### `audit/audit_ledger.py`

```python
from deep_ion.agents_engine.domain.protocols import AuditPort
from .decision_record import DecisionRecord

AUDIT_COMMENT_PREFIX = "## AUDIT-LEDGER\n"

class AuditLedger:
    def __init__(self, publisher: AuditPort, issue_number: int) -> None:
        self._publisher = publisher
        self._issue_number = issue_number

    def emit(self, record: DecisionRecord) -> None:
        # Serializa via record.model_dump_json()
        # Prefixo obrigatório: AUDIT_COMMENT_PREFIX
        # Formata como bloco de código JSON no comentário
        body = f"{AUDIT_COMMENT_PREFIX}```json\n{record.model_dump_json(indent=2)}\n```"
        self._publisher.emit(record)  # delega ao AuditPort (não acessa GitHub diretamente)
```

### Contrato de Evolução

Este schema é **imutável após o primeiro uso em produção**:
- ✅ Adicionar campo com `default` ou `default_factory` — permitido
- ❌ Remover campo existente — proibido (quebra leitores históricos)
- ❌ Alterar tipo de campo existente — proibido
- ❌ Alterar os Literal values de `DecisionType` sem aprovação T2/T3

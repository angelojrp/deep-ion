### Objetivo

Implementar os modelos de domínio puros (Pydantic frozen) e os Protocols de porta (DI) sem qualquer import de I/O, SDK ou infraestrutura. Este é o núcleo imutável da arquitetura.

---

### Regra Absoluta

> `domain/` **não pode importar nada** de `infrastructure/`, `providers/` ou `audit/`. O fluxo de import é sempre de fora para dentro: `infrastructure` → `domain`, nunca o contrário.

---

### Especificação Técnica

#### `domain/models.py`

```python
from pydantic import BaseModel, ConfigDict

class IssueData(BaseModel):
    model_config = ConfigDict(frozen=True)
    number: int
    title: str
    body: str
    labels: tuple[str, ...]  # imutável
    author: str
    state: str  # "open" | "closed"

class CommentData(BaseModel):
    model_config = ConfigDict(frozen=True)
    id: int
    body: str
    author: str
```

#### `domain/protocols.py`

```python
from typing import Protocol, runtime_checkable
from .models import IssueData

@runtime_checkable
class GitHubPort(Protocol):
    def get_issue(self, issue_number: int) -> IssueData: ...
    def post_comment(self, issue_number: int, body: str) -> None: ...
    def add_label(self, issue_number: int, label: str) -> None: ...
    def remove_label(self, issue_number: int, label: str) -> None: ...
    def find_comment_by_prefix(self, issue_number: int, prefix: str) -> str | None: ...

@runtime_checkable
class AuditPort(Protocol):
    def emit(self, record: "DecisionRecord") -> None: ...
    # DecisionRecord importado via TYPE_CHECKING para evitar circular import com audit/
```

---

### Import Circular — Tratamento

`AuditPort` referencia `DecisionRecord` (definido em `audit/`). Para evitar import circular:

```python
from __future__ import annotations
from typing import TYPE_CHECKING
if TYPE_CHECKING:
    from deep_ion.agents_engine.audit.decision_record import DecisionRecord
```

# UC-005: Visualizar e Editar Tenant

**Módulo:** `tenants`  
**Classificação de Impacto:** T1  
**Prioridade:** `Must`  
**Esforço relativo:** `M`  
**Dependências:** UC-001 (o usuário chega a esta tela a partir da listagem)  
**Regras de Negócio Acionadas:** RN-001, RN-002, RN-009  
**Ator Principal:** Admin da Plataforma  
**Pré-condições:**
- Admin da Plataforma autenticado com JWT válido contendo `papel = ADMIN_PLATAFORMA`
- Tenant selecionado a partir da listagem (UC-001)

**Pós-condições de Sucesso (visualização):**
- Dados completos do tenant exibidos na tela de detalhe: `tenant_id`, `nome`, `slug` (readonly), `status` (via toggle), `criado_em`, `membros_ativos`

**Pós-condições de Sucesso (edição de `nome`):**
- `nome` atualizado no banco de dados
- Toast de sucesso exibido; campo exibe o novo valor salvo

**Pós-condições de Falha (edição):**
- Alteração não persistida no banco de dados
- Banner de erro inline exibido próximo ao campo `nome`; campo retorna ao valor anterior

---

## Tabela de definições de atributos

| Descrição | Nome | Tipo | Domínio | Obrigatoriedade |
|---|---|---|---|---|
| Identificador único do tenant | tenant_id | ULID | Gerado pelo sistema; exibido como readonly | Sim (readonly) |
| Nome comercial do tenant | nome | string | Máx. 100 caracteres; único campo editável nesta tela | Sim |
| Identificador URL-friendly do tenant | slug | string | Imutável após criação (RN-002); exibido como readonly | Sim (readonly) |
| Estado atual do tenant | status | enum | ATIVO (toggle LIGADO) / INATIVO (toggle DESLIGADO) | Sim |
| Data e hora de criação do registro | criado_em | datetime (UTC) | Gerado pelo sistema; readonly | Sim (readonly) |
| Quantidade de membros com `status_convite = ATIVO` | membros_ativos | inteiro (≥ 0) | Calculado pelo sistema; readonly; gerenciado exclusivamente pelo Admin de Tenant (RN-009) | Sim (readonly) |

---

## Análise curta

- **Necessidade:** O Admin da Plataforma precisa consultar o cadastro completo de um tenant e corrigir o `nome` comercial quando necessário, além de acessar as ações de desativação/reativação a partir de um ponto centralizado (tela de detalhe).
- **Escopo:** Formulário misto — leitura de todos os campos + edição exclusiva do campo `nome`. O `slug` é exibido mas imutável (RN-002). A contagem de `membros_ativos` é informativa; o gerenciamento de membros é exclusivo do Admin de Tenant via módulo RM-F03 (RN-009). As ações de desativar (UC-003) e reativar (UC-004) são acessíveis a partir desta tela.
- **Ambiguidades:** Nenhuma aberta.
- **RNs relevantes:** RN-001 (`tenant_id` imutável e readonly); RN-002 (`slug` imutável após criação, exibido como readonly); RN-009 (Admin da Plataforma visualiza apenas a contagem de membros — não gere membros).

---

## Fluxo Principal — Visualização

| Passo | Ator | Ação | Sistema |
|---|---|---|---|
| 1 | Admin da Plataforma | Clica em "Visualizar" ou no nome do tenant na listagem (UC-001) | — |
| 2 | — | — | Emite `GET /tenants/{tenant_id}` com JWT no header `Authorization` |
| 3 | — | — | Retorna todos os campos do tenant: `tenant_id`, `nome`, `slug`, `status`, `criado_em`, `membros_ativos` |
| 4 | — | — | Renderiza tela de detalhe com: campos readonly (`tenant_id`, `slug`, `criado_em`), campo editável (`nome`), toggle de status, contagem de `membros_ativos` (readonly), botões de ação de zona de perigo (Desativar/Reativar via UC-003/UC-004) |

---

## Fluxo Principal — Edição do Nome

| Passo | Ator | Ação | Sistema |
|---|---|---|---|
| 1 | Admin da Plataforma | Clica no campo `nome` ou no ícone de lápis ao lado dele | — |
| 2 | — | — | Habilita o campo `nome` para edição inline |
| 3 | Admin da Plataforma | Edita o valor do campo `nome` | — |
| 4 | Admin da Plataforma | Clica em "Salvar" ou pressiona Enter | — |
| 5 | — | — | Valida que `nome` não está vazio e respeita o limite de 100 caracteres |
| 6 | — | — | Emite `PATCH /tenants/{tenant_id}` com payload `{ "nome": "{novo_nome}" }` |
| 7 | — | — | Persiste o novo valor no banco de dados |
| 8 | — | — | Exibe toast de sucesso _"Nome atualizado com sucesso"_; campo volta para modo readonly exibindo o novo nome |

---

## Fluxos Alternativos

**FA-01: Navegar para desativação (danger zone)** — Bifurca no Passo 4 do FP-Visualização  
O Admin da Plataforma clica no botão "Desativar Tenant" da danger zone ou move o toggle de LIGADO para DESLIGADO. O sistema inicia UC-003 (Modal de Confirmação).

**FA-02: Navegar para reativação** — Bifurca no Passo 4 do FP-Visualização (somente quando `status = INATIVO`)  
O Admin da Plataforma move o toggle de DESLIGADO para LIGADO. O sistema inicia UC-004 (reativação imediata, sem modal).

---

## Fluxos de Exceção

**FE-01: Erro ao carregar dados do tenant** — Bifurca no Passo 2 do FP-Visualização  
**Gatilho:** O endpoint `GET /tenants/{tenant_id}` retorna erro HTTP (4xx ou 5xx) ou timeout.  
**RN Violada:** — (falha de infraestrutura)  
**Resposta do Sistema:** Exibe banner de erro com mensagem _"Não foi possível carregar os dados do tenant. Tente novamente."_ e botão "Voltar para a listagem". A tela de detalhe não é renderizada.

**FE-02: Erro ao salvar edição de nome** — Bifurca no Passo 6 do FP-Edição  
**Gatilho:** O endpoint `PATCH /tenants/{tenant_id}` retorna erro HTTP ou timeout.  
**RN Violada:** — (falha de infraestrutura)  
**Resposta do Sistema:** Exibe banner de erro inline próximo ao campo `nome`: _"Falha ao salvar. Tente novamente."_ O campo permanece em modo de edição com o valor que o usuário digitou preservado. O banco de dados não é alterado.

---

## Invariantes

- O campo `slug` é SEMPRE exibido como readonly nesta tela; não deve existir controle de edição para ele (RN-002).
- O campo `tenant_id` é SEMPRE exibido como readonly (RN-001).
- O Admin da Plataforma pode visualizar `membros_ativos` mas NÃO pode iniciar nenhuma ação de convite, remoção ou gerenciamento de membros a partir desta tela (RN-009).

## Critérios de Aceitação (Given/When/Then)

**Cenário 1 — Caminho Feliz (Visualização):**
```gherkin
Given que o Admin da Plataforma está autenticado com papel = ADMIN_PLATAFORMA
  And o tenant "Acme Corp" (slug: "acme-corp", status: ATIVO, membros_ativos: 3) existe no sistema
When o Admin da Plataforma clica em "Visualizar" na linha do tenant "Acme Corp"
Then o sistema exibe a tela de detalhe com todos os campos
  And o campo slug exibe "acme-corp" sem controle de edição
  And o campo tenant_id exibe o ULID sem controle de edição
  And o toggle de status está na posição LIGADO (ATIVO)
  And o contador de membros ativos exibe "3" sem botão de gerenciamento
```

**Cenário 2 — Caminho Feliz (Edição de Nome):**
```gherkin
Given que o Admin da Plataforma está na tela de detalhe do tenant "Acme Corp"
When o Admin da Plataforma edita o nome para "Acme Corporation"
  And clica em "Salvar"
Then o sistema atualiza o nome para "Acme Corporation" no banco de dados
  And o toast "Nome atualizado com sucesso" é exibido
  And o campo nome exibe "Acme Corporation" em modo readonly
```

**Cenário 3 — FE-02 (Erro ao Salvar):**
```gherkin
Given que o Admin da Plataforma editou o nome para "Acme Corporation"
  And o backend está indisponível
When o Admin da Plataforma clica em "Salvar"
Then o sistema exibe "Falha ao salvar. Tente novamente." inline próximo ao campo
  And o campo nome permanece em modo de edição com o valor "Acme Corporation"
  And o banco de dados não é alterado
```

**Cenário 4 — FA-01 (Navegar para Desativação):**
```gherkin
Given que o Admin da Plataforma está na tela de detalhe do tenant "Acme Corp" com status = ATIVO
When o Admin da Plataforma clica em "Desativar Tenant" na danger zone
Then o sistema exibe o Modal de Confirmação de desativação (UC-003, Passo 2)
```
